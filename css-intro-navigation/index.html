Create Navigational Links Within A Website

There are several techniques to navigate within a website. The semantic tag for navigation is the <nav> tag. You can navigate through your website by using relative addressing. For our examples we will use the unordered list tag(<ul>).

  • page to page within the same folder
  • page to page between folders
  • staying on the current page
  • going to a specific place on the current page(on page navigation)

Navigational menus are placed in your <header> semantic area. They may also be placed in your <footer> area as a sitemap. The nav tag like other semantic tags should have a <hx> tag. HOWEVER, this header is optional. If you omit the header tag youu will recieve a documant outliner error: untitled section. Until you are shown how to avoid this error message, it will be acceptable on yourdocument outline.

CSS Coding Styles Summary

Browser Defaults (No CSS) You have no control over the styling
Inline This is for a single specific element (tag) instance. Your css will be specified in a parameter within the element. This css affects only the element tag that contains the specification. style="rule; [rule; ...]" <p style="color:red; width:50px;">...</p>
Embedded The specification of css as embedded happens in your <head> area. This ccs affects all similar elements on the web page where the css is coded <style> selector{rule; [rule; ...]} [selector{rule; [rule; ...]} ...]</style> <style> p {color:red; width:50px;}
h1 {font-style:italic}</style>
Link (StyleSheet) This is the preferred method of doing css. You place your css in a seperate physical file. This file can then be used on ANY web page in your website. The link to this file will be done in your <head> area. The css in this file is similar to embedded EXCEPT there is no style tag. selector{rule; [rule; ...]} [selector{rule; [rule; ...]} ...] p {color:red; width:50px;}
h1 {font-style:italic}
